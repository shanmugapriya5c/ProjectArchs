Architecture Layers

▲ Inbound Layer

  • Spring MVC REST Controller
  • Bean validation (@Valid)
  • MapStruct DTO mapping
  • Persist to Oracle (PENDING)
  • Return HTTP 200 immediately

▶ Integration Layer

  • Spring Integration DirectChannel
  • DynamicRecipientListRouter
  • One MessageHandler per operation
  • ServiceHelper (cross-cutting)
  • Spring Retry on proxy calls

▼ Outbound Layer

  • Generated OpenAPI client stubs
  • Hand-written proxy wrappers
  • Apache HttpClient5 (SSL-aware)
  • Reference data invoker (cached)
  • EhCache 3 (cache-aside)
Key Patterns
📩 Async Request-Reply (Outbox) Caller gets instant HTTP 200. Oracle acts as durable outbox — batch picks up and processes async.
🔄 Property-Driven Message Routing DynamicRecipientListRouter resolves target channel at runtime from externalized property — no redeploy.
⚡ Priority-Based Parallel Processing Batch groups by priority. ExecutorService processes concurrently — high-priority never starved.
🛡 Startup Recovery (Fail-Safe) CommandLineRunner marks IN_PROGRESS as FAILED on restart — prevents silent data loss.
💾 Cache-Aside (EhCache 3) Reference data cached via Spring Cache abstraction — eliminates redundant round-trips.
⇄ Proxy Abstraction Layer Generated OpenAPI stubs wrapped with SSL, retry, error mapping — insulates from generated code churn.
🌎 Single-Artifact Multi-Environment One JAR, all envs. Spring Profiles + Jasypt-encrypted credentials externalize config — zero env-specific builds.
